## 📝 Notas Técnicas

### Migração SharedPreferences → SQLite
- Migração automática executada no primeiro launch após update
- Dados de SharedPreferences são preservados durante migração
- Chaves de app settings (onboarding_complete, theme_mode) permanecem em SharedPreferences
- Quando usuário faz sign-out, dados SQLite ficam no dispositivo (pode ser limpo manualmente)

### Schema Design
- `meals` tem campo `sync_status` para tracking offline-first
- `sync_queue` substitui o antigo `pending_sync` do SharedPreferences
- `food_cache` tem TTL implícito via `cached_at` (limpeza no startup)
- Todas as tabelas usam TEXT para datas (ISO-8601) — mais fácil de query

### Performance
- Índices criados em: date, sync_status, user_id, search_key
- Meal foods usam CASCADE DELETE — deletar meal remove foods automaticamente
- Food cache limitado a 20 resultados por search_key
